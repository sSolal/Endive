Getting Started with Endive

This tutorial will guide you through installing Endive and constructing your first formal proofs using term rewriting.

Installation

Requirements

  • Python 3.8 or higher
  • Git (for cloning the repository)

Installing Endive

Clone the repository and install Endive:

# Clone the repository
git clone https://github.com/sSolal/Endive.git
cd endive

# Install using pip
pip install -e .

Running Endive

Start the interactive command-line interface:

# Start Endive CLI
python main.py

# Start with debug mode for verbose output
python main.py --debug

You should see a prompt where you can enter Endive commands. Type :help for help or :exit to quit.

Core Concepts

Objects and Rewritings

In Endive, everything is built from objects. An object can be:

  • A symbol: A, plus, zero
  • A term: f(x), plus(2, 3)
  • A hole (variable): #x, #n
  • A rewriting: A => B, x = y
  • A composition: f | g (compose f with g)

A rewriting transforms one object into another. Rewritings are written as: 

#x => #y     // Simple implication
#x = #y      // Equality (bidirectional)
#x <=> #y    // Equivalence

Buildability

The key question Endive answers is: "Can I build object X from the given axioms and rewritings?" An object is buildable if:

  1. It's already in the context (an axiom)
  2. It's a composition of two buildable objects
  3. It's a rewriting where the output is buildable with the input added to context

Pattern Matching

Holes in patterns match any object. For example, the pattern #x => #y matches:

  • A => B (with #x = A, #y = B)
  • plus(2, 3) => 5 (with #x = plus(2, 3), #y = 5)

Your First Proof

Example: Modus Ponens

Let's prove a simple logical argument: if we know A and A => B, we can derive B.

# Declare our assumptions
Axiom A
Axiom h, A => B

# State what we want to prove
Goal B

# Build the proof
Start A
Use h

Explanation:

  • Axiom A: We assume A is true
  • Axiom h, A => B: We have a rewriting named h that transforms A into B
  • Goal B: We want to prove B
  • Start A: Begin the proof with A
  • Use h: Apply the rewriting h to get B

If successful, Endive will confirm that B is buildable!

Example: Composing Rewritings

We can compose multiple rewritings together using the composition operator |:

Axiom A
Axiom h1, A => B
Axiom h2, B => C
Axiom h3, C => D

Goal D
Start A
Use h1 | h2 | h3

This applies h1, then h2, then h3 in sequence: A → B → C → D.

Working with Patterns

Rewritings with Holes

Holes (written as #name) act as variables that can match any object:

Axiom modus_ponens, (#x & (#x => #y)) => #y
Axiom A
Axiom A => B

Goal B
Start (A & (A => B))
Use modus_ponens

Here, #x matches A and #y matches B.

Example: Symmetry of Equality

Axiom sym, #x = #y => #y = #x
Axiom eq, 2 = 3

Goal 3 = 2
Start (2 = 3)
Use sym

The pattern #x = #y matches 2 = 3, producing 3 = 2.

Reduction and Computation

Endive can also reduce objects by applying computational rewritings:

Axiom plus_zero, plus(#x, zero) = #x
Axiom plus_succ, plus(#x, S(#y)) = S(plus(#x, #y))

Reduce plus(2, 3)

The Reduce command simplifies the object by repeatedly applying matching rewritings. This enables both symbolic computation and proof verification in the same framework.

Checking Objects

Use Check to verify if an object is buildable without proving it step-by-step:

Axiom A
Axiom h, A => B

Check B  # Returns: buildable

Functoriality

One of Endive's powerful features is functoriality: the ability to apply rewritings inside nested structures. 

Axiom le_3_4, 3 <= 4
Axiom functorial_mult, #x <= #y => (#x * #z) <= (#y * #z)

Goal (3 * 2) <= (4 * 2)
Start (3 <= 4)
Use functorial_mult

The rewriting functorial_mult allows us to multiply both sides of an inequality, demonstrating how proofs can be lifted into more complex contexts.

Defining and Importing

Defining Aliases

Create named definitions for reusable objects using Define:

// Define a simple conjunction builder
Define def_and, #A => #B => and(#A, #B)
Axiom def_and

// Use it in proofs
Goal and(P, Q)
Intro
Intro
By def_and

Importing Modules

Import definitions from other files using Using:

// Import logic definitions
Using logic

// Import arithmetic axioms
Using arithmetic

// Now you can use definitions from both modules
Goal forall(N, #n -> (2 * #n) = (#n + #n))

The Using directive loads axioms and definitions from .end files, making it easy to build libraries of reusable proofs.

Tips for Effective Proving

  • Start simple: Begin with basic axioms and rewritings, then build complexity gradually
  • Use meaningful names: Name your axioms and rewritings descriptively (e.g., trans for transitivity)
  • Check intermediate steps: Use Check to verify that intermediate objects are buildable
  • Debug with --debug: Run Endive in debug mode to see detailed matching and reduction steps
  • Experiment with reduction: Use Reduce to explore how objects simplify
  • Leverage functoriality: Don't repeat proofs for similar structures—use functorial rewritings
