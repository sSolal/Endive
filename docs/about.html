About Endive

Endive is an open source proof assistant that explores formal verification through the lens of term rewriting.

Project Status

Endive is currently in early development (version 0.1.0). The project has a solid foundation with:

  • Core rewriting engine
  • Pattern matching and reduction
  • Buildability checking
  • Interactive command-line interface
  • Modular helper system for extensibility

While functional, Endive is actively evolving. We welcome feedback, bug reports, and contributions as we continue to develop and refine the system.

Getting Started as a Contributor

  1. Fork the repository on GitHub
  2. Clone your fork to your local machine: 
    git clone https://github.com/your-username/Endive.git
cd Endive
  3. Create a branch for your changes: 
    git checkout -b feature/your-feature-name
  4. Make your changes and test them thoroughly
  5. Run tests to ensure nothing breaks: 
    python main.py --test
  6. Commit your changes with clear, descriptive messages: 
    git commit -m "Add feature: brief description"
  7. Push to your fork: 
    git push origin feature/your-feature-name
  8. Submit a pull request on GitHub with a description of your changes

Architecture

Endive is built with a clean, modular architecture that separates concerns:

Core Layer

The src/core/ directory contains the fundamental data structures and operations:

  • objects.py: Term, Rewriting Rule, Composition, and Hole classes
  • operations.py: Pattern matching, rewriting application, composition, and reduction
  • buildability.py: Buildability checking logic

Engine Layer

The src/engine/ directory implements the parsing and execution pipeline:

  • parser.py: Tokenization and parsing of Endive syntax
  • pipeline.py: Helper orchestration and hook management
  • engine.py: Main execution engine
  • display.py: Output formatting and visualization
  • helpers/: Modular helper system for extensions

Application Layer

The src/app/ directory provides user interfaces:

  • cli.py: Interactive command-line interface with REPL

This layered architecture ensures that the core rewriting logic remains simple and correct, while extensibility is achieved through the helper system.

Philosophy

Endive is guided by several key principles:

Simplicity

The core of Endive is intentionally minimal. Complex features are built on top of simple primitives through the helper system, keeping the foundational logic clean and verifiable.

Rewriting First

Rather than building on type theory or set theory, Endive embraces term rewriting as the primary mechanism for both computation and proof. This provides a unique perspective on formal verification.

Extensibility

The modular helper system allows users to extend Endive without modifying core code. This enables experimentation with different proof strategies and features.

Accessibility

Endive aims to be approachable for newcomers to formal verification while remaining powerful enough for serious mathematical work. Clear documentation and simple syntax support this goal.
