Why Proof Assistants Matter

In mathematics and computer science, formal verification provides absolute certainty that a proof or program is correct. Proof assistants are powerful tools that help us construct and verify these formal proofs.

Proof assistants enable mathematicians, logicians, and computer scientists to:

  • Verify complex proofs with machine-checked certainty, eliminating subtle errors that might escape human review
  • Build reliable software by formally proving that programs meet their specifications
  • Explore new mathematics by experimenting with formal systems and discovering patterns
  • Teach rigorous reasoning by making logical steps explicit and transparent

As mathematical proofs grow increasingly complex and software becomes more critical to society, the importance of mechanical verification continues to grow.

Rewriting as a Foundation for Reasoning

Term rewriting is a simple yet powerful computational model that naturally captures the essence of logical reasoning.

At its core, rewriting is about transformation: given an object (an expression) and a set of rewritings, we can systematically transform that object by applying matching rewritings. This process mirrors how we reason:

"If I know A and I know that A implies B, then I can conclude B."

Rewriting systems offer several advantages for formal reasoning:

Simplicity

Rewritings are intuitive: replace a pattern with another object. This simplicity makes the system accessible while remaining powerful.

Composability

Complex proofs emerge from composing simple rewriting steps, mirroring how mathematicians build arguments incrementally.

Computation and Logic

Rewriting unifies computation (evaluating expressions) with logical reasoning (deriving conclusions), following the Curry-Howard correspondence.

Functoriality

Rewritings can be applied within nested contexts, enabling modular reasoning about complex structures.

What is Endive?

Endive is a minimalist proof assistant where you transform mathematical objects using rewriting rules.

With Endive, you can:

  • Build formal proofs by composing simple rewriting steps
  • Verify mathematical statements through buildability checking
  • Compute with symbolic expressions using pattern matching and reduction
  • Explore formal systems from logic to arithmetic to your own custom theories

Unlike proof assistants based on type theory or set theory, Endive focuses on term rewriting as the primary mechanism for both computation and proof construction. This approach offers a unique perspective on formal verification that's both simple to understand and powerful in practice.

Key Features

  • Pattern matching with holes: Define rewritings using patterns with variables (holes) that match arbitrary objects
  • Buildability checking: Verify whether an object can be constructed from axioms and rewritings in a given context
  • Reduction and computation: Simplify objects by applying rewritings, enabling both symbolic manipulation and evaluation
  • Functoriality: Apply rewritings within nested structures, allowing proofs to compose naturally
  • Syntactic sugar: Use natural notation for numbers, quantifiers, and logical operators while maintaining a simple core
  • Extensible architecture: Modular helper system allows adding new capabilities while preserving the core engine

A Quick Example

Here's how you define and use a simple rewriting rule in Endive:

// Define a rule that constructs logical conjunction
Define def_and, #A => #B => and(#A, #B)

// Declare it as an axiom
Axiom def_and

// Now you can use it to build proofs
Goal and(P, Q)
Intro  // Introduce #A
Intro  // Introduce #B
By def_and

This example shows holes (#A, #B) that act as pattern variables, a goal-driven proof where we work backwards from what we want to prove, and how rewriting rules transform one expression into another. The Intro directive introduces hypotheses, and By applies our rewriting rule to complete the proof.
